i try to avoid porting complex configs to nixos to maintain distro portability, so i dump them here

neovim is a special headache where i [mkOutOfStoreSymlink inside the home-manager module](https://github.com/graevy/nixos/blob/64bce2782d8a0f905350cbc684e70cec6e70830d/home-manager-a.nix#L18C1-L18C105). this fixes mason/treesitter/hardcoding binaries with a store object while providing lazy-nvim a *writeable* symlink. i haven't found a better way yet. i'm also attempting to use neovim's native config as much as possible; nixos isn't adequate here. config is only stored in nixos insofar as `fzf` and `lazygit` etc. are being managed by nix instead of `luarocks` under lazy.
